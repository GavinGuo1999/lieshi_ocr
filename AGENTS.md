# AGENTS.md

你是 lieshi_ocr 项目的代码执行助手。

## 项目核心原则

1. 当前可信 Excel 基线是 `英名录25版-祁县-二审_v4.xlsx`，不是 v5。
2. `英名录25版-祁县-二审_v5.xlsx` 是待审计候选产物，不能作为默认业务基线。
3. `old_code/` 是第一次提取历史代码，只能作为 legacy 参考。
4. `new_code/` 是第二次提取代码，算法更接近未来主线，但仍需拆分重构。
5. 任何重构都不能改变已有业务输出，除非任务明确要求。
6. 所有 Excel 修改必须先 dry-run，禁止直接覆盖原始 Excel。
7. 原始扫描 PDF、大型中间产物、真实 Excel 基线默认不提交到 Git。
8. 每次任务只能做一个小范围改动。
9. 不能删除 legacy 代码，除非任务明确要求。
10. 不能引入重型新依赖，除非先说明理由。

## 目录约定

- `data/scan/{batch}/`：原始扫描 PDF。
- `data/work/{batch}/`：中间裁剪、OCR、MinerU 结果。
- `data/output/{batch}/`：输出报告和候选 Excel。
- `data/private/`：真实基线 Excel 和不应公开的数据。
- `src/lieshi_ocr/`：后续正式代码目录。
- `scripts/legacy/`：后续历史脚本目录。
- `old_code/`：当前保留的第一轮历史代码。
- `new_code/`：当前保留的第二轮候选主线代码。
- `tests/fixtures/`：小型脱敏测试数据。

## Codex 每次完成任务必须输出

1. 修改了哪些文件。
2. 为什么这样改。
3. 是否改变业务逻辑。
4. 如何运行。
5. 运行了哪些测试。
6. 是否有失败测试。
7. 下一步建议。
