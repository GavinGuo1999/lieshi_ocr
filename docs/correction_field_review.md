# 修正内容字段提取核对

## 目标

本阶段把扫描件“修正内容及理由”中的每个“字段补充完善为……理由……”项目提取出来，
按照 v4 的字段结构提供人工核对。

重点是正确归类字段并完整保留业务值。字段名中的固定 OCR 错字可以容错，但姓名、地名、
单位、职务和事迹正文不得自动猜测或改写。

## v4 对照字段

核对表使用 v4 的以下正式字段作为对照：

- 编号
- 姓名
- 性别
- 籍贯
- 出生时间
- 参加革命（工作）时间
- 政治面貌
- 生前单位及曾任职务
- 牺牲时间、地点及简要事迹

民族、牺牲时间、牺牲地点、牺牲原因、事迹和安葬地仍分别提取，便于人工核对；本阶段
不把这些字段自动合成为 K 列。

## 字段标签容错

容错只作用于“补充完善为”前的固定字段标签。例如字段首字漏识别或“牺牲”被识别为
近似字时，可以归入对应标准字段。

如果字段标签无法识别，但理由中只出现一个明确的“原某字段”提示，可以根据该理由归类，
并记录 `field_inferred_from_reason`。理由不唯一时不做猜测。

所有自动归类都保留：

- OCR 原字段名
- 系统归类字段
- 完整提取值
- 完整理由
- 归类提示

## 10 人核对文件

本地文件：

`data/output/20260626/10人v4字段提取核对表.xlsx`

工作簿包含四个 sheet：

1. `人员概览`：10 人的 v4 当前值和扫描提取值并排展示。
2. `字段核对`：每个修正项目一行，黄色列用于填写人工确认值、状态和备注。
3. `原始正文`：完整 MinerU 原文和本地裁剪路径。
4. `使用说明`：核对边界和操作步骤。

当前 10 人均按编号和姓名匹配到 v4，共提取 64 条修正项目，未识别字段为 0。

## 安全边界

- 不修改 v4。
- 不读取或使用 v5 作为基线。
- 不生成、压缩或写入 K 列候选。
- 不自动修正单位、职务、姓名、地名或事迹正文。
- 不执行 Excel dry-run 或 apply。
- 真实核对工作簿位于被 Git 忽略的 `data/output/`，不得提交或公开上传。
