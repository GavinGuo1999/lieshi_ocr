# 修正内容 OCR 人工核对

## 目的

真实扫描件的首要审核对象是完整的“修正内容及理由”，不是自动生成的 K 列事迹候选。

本阶段只完成以下工作：

1. OCR 识别编号和姓名，用于确认扫描件身份。
2. 保留完整的修正内容及理由 OCR/MinerU 文本。
3. 由人工对照原扫描件修正错字、漏字和断行。

在人工核对完成前，不从 OCR 文本生成 K 列候选，不修改 v4，也不执行 Excel apply。

## 10 条核对样本

当前本地样本来自 `data/work/20260626/text/text_manifest.json` 中前 10 个同时具有
`code`、`name` 和 `correction` 文本的记录。

本地生成文件：

`data/output/20260626/10条OCR修正内容核对表.xlsx`

该工作簿包含真实姓名和修正内容，位于被 Git 忽略的 `data/output/`，不得提交或公开上传。

## 核对表字段

- OCR 编号 / 人工核对编号
- OCR 姓名 / 人工核对姓名
- OCR 修正内容及理由 / 人工核对后修正内容及理由
- 核对结论
- 备注
- 原扫描 PDF 和修正内容裁剪 PDF 的本地路径

人工列预填 OCR 原文，便于直接修改；核对结论默认为“未核对”。

## 当前边界

- 不生成或压缩 K 列内容。
- 不判断 T 列的最终删除时点。
- 不修改可信基线 `英名录25版-祁县-二审_v4.xlsx`。
- 不使用 v5 作为基线。
- 不提交真实 PDF、Excel、OCR 文本或核对结果。
